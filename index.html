  • CSS is a stylesheet language used to describe the presentation of a document written in HTML or XML

CSS Syntax




Selector

This is the HTML element name at the start of the ruleset. It defines the element(s) to be styled . To style a different element, change the selector.

Declaration

This is a single rule like color: red;. It specifies which of the element's properties you want to style.

Properties

These are ways in which you can style an HTML element. In CSS, you choose which properties you want to affect in the rule.

Values

To Right Property . To correct Values. For Eg. color:"blue"

What is MangoDB ?

  • MongoDB is a non-relational document database that provides
  • It Support for JSON-like storage
  • The MongoDB database has a flexible data model that enables you to store unstructured data.
  • It provides full indexing support, and replication with rich and intuitive APIs.

Key Aspects of MangoDB

  • Documents : The Documents in the Database
  • collections : Grouping Documents
  • Replica sets : Ensuring Avalibility
  • Shrading : Scalibilty to growth of Massive Data
  • Indxes : Incresing the query speed
  • Aggeration Pipelines : Fast Data Flows
  • Programming Language : MangoDB

Documents : The Documents in the Database

MongoDB stores data as JSON documents.The document data model maps naturally to objects in application code, making it simple for developers to learn and use.The fields in a JSON document can vary from document to document. Compare that to a traditional relational database table, where adding a field means adding a column to the database table itself and therefore to every record in the database.

\

collections : Grouping Documents

In MongoDB, a collection is a group of documents.If you are familiar with relational databases, you can think of a collection as a table. But collections in MongoDB are far more flexible. Collections do not enforce a schema, and documents in the same collection can have different fields.Each collection is associated with one MongoDB database. To show which collections are in a particular database, use the command listCollections.

Replica sets : Ensuring Avalibility

An important way to ensure high availability is by keeping more than one copy of your data. With MongoDB, high availability is built right into the design. When you create a database in MongoDB, the system automatically creates at least two more copies of the data, referred to as a replica set. A replica set is a group of at least three MongoDB instances that continuously replicate data between them, offering redundancy and protection against downtime in the face of a system failure or planned maintenance.

Shrading : Scalibilty to growth of Massive Data

A modern data platform needs to be able to handle very fast queries and massive datasets using ever bigger clusters of small machines. Sharding is the term for distributing data intelligently across multiple machines.How does sharding work in MongoDB? MongoDB shards data at the collection level, distributing documents in a collection across the shards in a cluster. The result is a scale-out architecture that supports even the largest applications.

Indxes : Incresing the query speed

Indexes support the efficient execution of queries. MongoDB offers a variety of different indexing strategies, including compound indexes on multiple fields. Chosen carefully, indexes speed up queries because queries scan the index instead of reading every document in the collection.There is still work to do to analyze which queries could benefit from adding an index. One tool that does this analysis for you is Performance Advisor, which analyzes queries and suggests indexes that would improve query performance.

Aggeration Pipelines : Fast Data Flows

MongoDB offers a flexible framework for creating data processing pipelines called aggregation pipelines. It features dozens of stages and over 150 operators and expressions, enabling you to process, transform, and analyze data of any structure at scale. One recent addition is the Union stage, which flexibly aggregate results from multiple collections.

Programming Language : MangoDB

What languages can you use with MongoDB? The list of supported languages includes Node.js, C, C++, C#, Go, Java, Perl, PHP, Python, Ruby, Rust, Scala, and Swift. The library for each language is actively maintained, which means that it is updated with new features, bug fixes, security patches, and performance enhancements.

Uses Of Node.JS

Key Features

  • Implemented in JavaScript.
  • Asynchronous Nature.
  • Event-Driven Architecture
  • Single-Threaded Working.
  • Scalability.
  • Multiple platform compatibility
  • Quick Data Streaming
  • Minimum Buffering

Implemented in Javascript

Node. js (Node) is an open source development platform for executing JavaScript code server-side. Node is useful for developing applications that require a persistent connection from the browser to the server and is often used for real-time applications such as chat, news feeds and web push notifications

Asynchronous Nature

Asynchronous means non-blocking. It does not wait to complete a task to go into another task. JavaScript is synchronous by nature. But there are a few things that have asynchronous behavior such as API calls, setTimeout, DOM events (click, mouse over, scroll …).

Event-Driven Architecture

Event-driven programming is used to synchronize the occurrence of multiple events and to make the program as simple as possible. The basic components of an Event-Driven Program are: A callback function ( called an event handler) is called when an event is triggered.

Single-Threaded Working

The execution of an entire task from beginning to end without interruption

Scalability

Node.js offers Easy Scalability. js uses a single thread to handle non-blocking I/O calls, which means that it'll take fewer resources for the application to accept concurrent connections compared to traditional approaches.

Multiple Platform Compatibility

Multi-platform—Cross-platform support allows you to create SaaS websites, desktop apps, and even mobile apps, all using Node. js

Quick Data Streaming

Streams are objects that let you read data from a source or write data to a destination in continuous fashion. In Node.js, there are four types of streams − Readable − Stream which is used for read operation.

Minimum Buffering

Generally, Buffer refers to the particular memory location in memory. Buffer and array have some similarities, but the difference is array can be any type, and it can be resizable. Buffers only deal with binary data, and it can not be resizable. Each integer in a buffer represents a byte. console.
